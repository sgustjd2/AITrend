## 🔥 오늘의 핵심 이슈

### 1. Claude Code 성능 저하 사후 분석 공식 공개 (Anthropic)
- **내용**: Anthropic이 3월~4월 중 발생한 Claude Code 성능 저하 원인을 공식 사후 분석(Postmortem)으로 공개
- **3가지 원인**:
  1. 3월 4일: Claude Code 기본 추론 노력(reasoning effort)을 `high → medium`으로 하향
  2. 3월 26일: 세션 중간에 추론 이력을 지속 삭제하는 버그 배포
  3. 4월 16일: 도구 호출 사이 응답을 25단어로 제한하는 시스템 프롬프트 추가
- **해결**: 4월 20일 3가지 모두 수정, 구독자 사용 제한도 초기화
- **논란**: 경쟁사 경영진들이 유출된 Claude Code 소스코드(3월 31일 npm에 512,000줄 3시간 노출)를 엔지니어링팀에 조용히 전달했다는 보도
- **시사점**: AI 서비스 품질 일관성이 기업 신뢰도의 핵심 이슈로 부상

### 2. GPT-5.5 정식 출시 및 "위임 가능한 AI" 패러다임 전환
- **출시일**: 4월 23일
- **핵심 컨셉**: 계획 → 도구 사용 → 검증 → 완료의 자율 순환 구조
- **강점 영역**: 에이전트 코딩, 컴퓨터 사용, 지식 업무, 초기 과학 연구
- **벤치마크**: Intelligence Index 60점(Gemini 57점 대비 우위), Terminal-Bench 2.0 82.7%, FrontierMath Tier 4 35.4%
- **가격**: 입력 $2/백만 토큰, 출력 $12/백만 토큰 (Gemini 대비 약 1.7배)
- **비전 기능**: 텍스트+이미지 멀티모달, Codex에서 컴퓨터 화면 읽기 지원

### 3. Claude Opus 4.7 프롬프트 엔지니어링 패러다임 변화
- **내용**: Claude Opus 4.7은 더 이상 애매한 지시를 스스로 보완해주지 않음
- **변화 핵심**: 과거 모델이 부족한 맥락을 넘겨짚어 채워줬다면, 이제는 입력된 텍스트를 정밀하게 그대로 실행
- **인사이트**: "AI가 고도화될수록 진짜 병목은 기술이 아니라 사람" — 정확한 지시문 작성 능력이 핵심 역량으로 부상
- **실무 적용**: 목표(달성할 것), 범위(건드릴 영역), 수용 기준(완료 조건) 3요소를 명확히 해야 함

### 4. Goldman Sachs 홍콩 직원 Claude 접근 차단
- **내용**: Goldman Sachs가 홍콩 소재 직원들의 Claude 접근 권한을 제거 (다른 AI 모델은 유지)
- **배경**: 금융·군사 영역에서 AI 사용에 대한 지정학적 규제 논쟁 확산
- **맥락**: 미국 군사 AI 활용 및 Google-국방부 계약이 동시에 주목받는 주간
