## 1. AI 기업 전략 & 제품

### Gemini Spark UI 추가 유출 — Agent / Chat 분리 구조

구글 I/O 공개 전 **Gemini Spark**의 UI 화면이 추가로 유출됐다.

| 항목 | 내용 |
|------|------|
| 모드 분리 | **Agent mode** / **Chat mode** 명확히 구분 |
| 도구 활용 | shell, browser 등 도구 사용 흔적 확인 |
| 실행 형태 | 사용자 노트북 직접 제어보다는 **클라우드 기반 작업 실행** 형태 |
| 기능 흐름 | 고급 도구 사용 + **Skills 생성** 흐름 포함 |
| 공개 범위 | 아직 제한된 사용자에게만 노출되는 초기 버전 |

> 포인트: Codex, Claude Code처럼 Gemini도 작업 실행 중심 에이전트 형태로 빠르게 이동 중이다. 구글 I/O에서 Android AI OS, 에이전트 흐름까지 공개 예정으로 큰 전환점이 될 가능성이 크다.

---

### Ollama + Codex App 지원 추가 — 오픈모델로 에이전트 UI 재활용

Ollama가 **Codex App** 지원을 추가했다.

| 항목 | 내용 |
|------|------|
| 연결 가능 모델 | Kimi, GLM, Qwen 등 오픈소스 모델 |
| 실행 방법 | `ollama launch codex-app` |
| 유지 기능 | Codex App의 에이전트 기능 + 브라우저 기반 작업 흐름 그대로 사용 |
| 비용 | OpenAI 토큰 비용 없이 Codex 경험 유지 가능 |

> 포인트: "좋은 에이전트 UI 위에 어떤 모델을 꽂느냐"가 더 중요해지는 흐름이다. 모델과 실행 환경이 분리되기 시작하는 신호다.

---

### Meta Ray-Ban Display 개발자 프리뷰 — 웹 기술로 스마트 글래스 UI

Meta가 Ray-Ban 스마트 글래스용 **개발자 프리뷰**를 공개했다.

| 항목 | 내용 |
|------|------|
| 개발 기술 | HTML, CSS, JavaScript (기존 웹 기술) |
| 입력 방식 | **Neural Band** 기반 제스처 입력 |
| 전략 방향 | 기존 iOS·Android·웹 개발 생태계를 그대로 스마트 글래스로 연장 |

> 포인트: 과거 VR·MR이 콘텐츠 부족으로 막혔다면, Meta는 기존 웹 개발자들을 그대로 끌고 오는 방향을 선택했다. AI 에이전트와 결합되면 "앱 실행 기기"가 아닌 "항상 착용한 AI 인터페이스"로 진화할 가능성이 있다.

## 2. AI 인사이트 & 사회 현상

### AI 혐오 실험 — 모네의 진품을 AI 그림이라 속이다

익명 아티스트 **SHL0MS**가 진행한 정교한 실험이 화제가 됐다.

**실험 경과:**

| 단계 | 내용 |
|------|------|
| 1단계 | "AI로 생성한 모네 스타일 그림, 진짜보다 어떤 점이 열등한지 설명해 달라" 게시 |
| 2단계 | 수많은 사람들이 색감·원근감·붓터치를 진지하게 비판 ("깊이감 없음", "터무니없이 모호") |
| 3단계 | 24시간 후 공개: 해당 이미지는 **실제 모네 원작의 크롭본** |
| 결말 | 대중에게 "열등하다"고 평가받은 모네 작품 조각을 NFT로 발행해 경매 판매 |

> 포인트: 'AI가 만들었다'는 꼬리표만으로 대중의 뇌가 무조건 오류를 찾으러 작동한다는 것을 실증했다. AI 콘텐츠 증가로 인해 인간의 신뢰 체계 자체가 흔들리고 있음을 보여주는 사례다.

### 신규 주목 — Gemma-4-E4B-it GGUF (Google / Unsloth)

**unsloth/gemma-4-E4B-it-GGUF** — Google Gemma 4 기반, MoE 경량화 VLM

| 항목 | 내용 |
|------|------|
| 아키텍처 | MoE (Mixture of Experts) — 8B 총 파라미터 / **4B 효과적 파라미터** |
| 이미지 처리 | Image-Text-to-Text (진정한 VLM) |
| 추론 도구 | llama.cpp, Ollama, vLLM, LM Studio, Jan 모두 지원 |
| 라이선스 | Apache-2.0 |
| 다운로드 | **1.24M** (매우 활발) |
| 커뮤니티 | 15개 토론 스레드 |

**빠른 실행 방법:**

```bash
# Ollama로 실행 (가장 간편)
ollama run hf.co/unsloth/gemma-4-E4B-it-GGUF:UD-Q4_K_XL

# llama.cpp 서버 실행 (OpenAI 호환 API)
llama-server -hf unsloth/gemma-4-E4B-it-GGUF:UD-Q4_K_XL
```

**Python으로 이미지 Q&A:**

```python
from llama_cpp import Llama

llm = Llama.from_pretrained(
    repo_id="unsloth/gemma-4-E4B-it-GGUF",
    filename="gemma-4-E4B-it-BF16.gguf",
)

llm.create_chat_completion(
    messages=[{
        "role": "user",
        "content": [
            {"type": "image_url", "image_url": {"url": "이미지_경로_또는_URL"}},
            {"type": "text", "text": "현재 이미지에 화재나 연기가 보이나요? 작업자가 안전고리를 착용하고 있나요?"},
        ],
    }]
)
```

> 포인트: MoE 덕분에 8B 파라미터지만 실질적으로 4B만 활성화되어 RTX 3070(8GB)에서도 Q4 양자화 시 구동 가능하다. Apache 2.0 라이선스라 상업적 활용도 자유롭다.

### MiniCPM-V-4.6 — 트렌딩 1위 유지, 다운로드 가속 중

**openbmb/MiniCPM-V-4.6** — 2026-05-14 대비 다운로드 5.7k 증가 (16.8k → 22.5k)

- 기반: SigLIP2-400M 인코더 + Qwen3.5-0.8B LLM
- VRAM: ~2GB (RTX 3070에서 가장 여유있게 구동)
- 배포: vLLM, SGLang, llama.cpp, Ollama 지원
- 라이선스: Apache-2.0
- 커뮤니티 채택 속도 가장 빠름 — 실시간 탐지 파이프라인 1순위 추천

### 화재·안전 감지 용도 추천 조합 (RTX 3070/4070)

| 목적 | 추천 모델 | GPU 요구 | 이유 |
