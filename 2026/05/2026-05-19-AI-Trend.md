### Google I/O 2026 — Gemini Omni 발표 (영상 AI 혁신)

Google이 I/O 2026에서 새로운 생성형 미디어 모델 **Gemini Omni**를 발표했다. "anything in, anything out" 방향으로, 이번에는 비디오부터 시작한다.

| 항목 | 내용 |
|------|------|
| 핵심 기능 | 사진·영상·음성·텍스트를 레퍼런스로 넣어 영상 생성 또는 기존 영상을 대화로 수정 |
| 특징 | AI 아바타 삽입, 스타일 레퍼런스 유지, 카메라 시점 변경, 대화 기반 반복 수정 |
| 비교 | GPT-4o의 이미지 "Nano Banana 모멘트"를 영상 영역으로 확장 |
| 주요 활용처 | 광고, 숏폼, 제품 영상, 크리에이터 영상 |
| 광고 혁신 포인트 | Netflix 등이 동일 원본 영상을 국가별로 인물·제품·자막·배경 다르게 변환 가능 |

> 포인트: 영상 제작에서 "처음 만드는 것"보다 "계속 수정하는 비용"이 핵심이었다. Omni는 이 수정 비용을 대화 한 번으로 줄인다는 발표에 가깝다.

### xAI 해커톤 우승작 — "Halftime" (광고 혁신)

시청 중인 영화·드라마의 맥락을 AI가 파악해 그 장면과 자연스럽게 이어지는 광고를 실시간 생성해 삽입하는 기술.

| 항목 | 내용 |
|------|------|
| 기술 | 영상 맥락 분석 → 맥락 연속 광고 자동 생성 |
| 특징 | 기존 광고처럼 흐름을 끊지 않고, 편집점이 매끄럽게 이어짐 |
| 의미 | 영상의 맥락 자체가 실시간 타겟팅 광고 트리거 |

> 포인트: 기술적·상업적 영리함. 구글 Gemini Omni의 영상 편집 능력과 결합하면 광고 산업의 전체 제작 파이프라인이 재편될 수 있다.

### 빠른 실행 — MiniCPM-V-4.6 (이미지 Q&A, 1B 초경량)

```python
from transformers import AutoProcessor, AutoModelForImageTextToText
from PIL import Image
import torch

processor = AutoProcessor.from_pretrained("openbmb/MiniCPM-V-4.6")
model = AutoModelForImageTextToText.from_pretrained(
    "openbmb/MiniCPM-V-4.6",
    torch_dtype=torch.float16,
    device_map="auto"
)

image = Image.open("작업현장.jpg")
messages = [
    {
        "role": "user",
        "content": [
            {"type": "image", "url": image},
            {"type": "text", "text": "현재 이미지에 화재나 연기가 보이나요? 작업자가 안전고리를 착용하고 있나요?"}
        ]
    }
]

inputs = processor.apply_chat_template(
    messages,
    add_generation_prompt=True,
    tokenize=True,
    return_dict=True,
    return_tensors="pt",
).to(model.device)

outputs = model.generate(**inputs, max_new_tokens=200)
print(processor.decode(outputs[0][inputs["input_ids"].shape[-1]:]))
```
