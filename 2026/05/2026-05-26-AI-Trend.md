> 포인트: 에이전트 AI는 스스로 반복 실행·검증·호출하며 토큰을 계속 소비한다. "AI를 쓰는 비용"이 "사람을 쓰는 비용"을 넘는 순간, 기업들은 경량화·효율화 방향으로 이동할 수밖에 없다.

### 빠른 실행 — GLM-OCR 1B (작업현장 안전 표지판 인식)

```python
from transformers import AutoProcessor, AutoModelForVision2Seq
from PIL import Image
import torch

processor = AutoProcessor.from_pretrained("zai-org/GLM-OCR")
model = AutoModelForVision2Seq.from_pretrained(
    "zai-org/GLM-OCR",
    torch_dtype=torch.float16,
    device_map="auto"
)

image = Image.open("작업현장.jpg")
messages = [
    {
        "role": "user",
        "content": [
            {"type": "image"},
            {"type": "text", "text": "이미지에 보이는 안전 표지판의 텍스트를 모두 읽어주세요. 화재 경고나 위험 표시가 있나요?"}
        ]
    }
]

prompt = processor.apply_chat_template(messages, add_generation_prompt=True)
inputs = processor(text=prompt, images=[image], return_tensors="pt").to(model.device)
outputs = model.generate(**inputs, max_new_tokens=300)
print(processor.decode(outputs[0][inputs["input_ids"].shape[-1]:]))
```

### 주목 — SkillOpt (Microsoft Research, 147표)

**SkillOpt**는 에이전트가 스킬을 실행한 후 피드백을 받아 스킬 자체를 자동으로 개선하는 "자기진화(Self-Evolving)" 프레임워크다.

> 포인트: 에이전트가 도구를 더 잘 쓰도록 스스로 학습하는 구조 — 반복 업무 자동화 패러다임과 직접 연결된다. Codex·Claude Code 같은 코딩 에이전트가 프로젝트별 최적 스킬을 자동 진화시키는 방향의 선행 연구다.

### 주목 — From Seeing to Thinking (VLM 인식-추론 분리)

VLM에서 시각 인식(이미지 이해)과 언어 추론(답변 생성)을 별도로 후처리 학습하면 전체 성능이 향상된다는 연구.

> 포인트: 화재·안전고리 감지 같은 특화 도메인 파인튜닝 시, "인식 단계"와 "판단 단계"를 분리해 학습하면 더 정확한 결과를 얻을 수 있다는 실용적 시사점.
