## 1. AI 기업 전략 & 제품

### ChatGPT 금융 OS — 계좌 연동으로 AI가 실생활 전체를 감싸다

오픈AI가 미국 Pro 사용자 대상으로 **금융 계좌 연동 기능**을 공개했다.

| 항목 | 내용 |
|------|------|
| 연동 대상 | 은행 계좌, Robinhood 등 금융 데이터 |
| 기능 | 소비 흐름 분석, 실제 거래 내역 기반 질의응답 |
| 확장 맥락 | 메일 → 캘린더 → 브라우저 → 코딩 → **금융 데이터**까지 연결 |
| 커뮤니티 반응 | "수백 개 핀테크 스타트업이 끝났다" vs "왜 금융 데이터를 AI 회사에 넘기냐" |

> 포인트: 방향성이 중요하다. AI 경쟁은 이제 "누가 더 많은 개인 데이터를 연결하고 사용자 맥락 전체를 이해하느냐"로 이동하고 있다. ChatGPT는 점점 사용자의 실제 삶 전체를 이해하는 에이전트로 확장 중이다.

---

### Codex 모바일 앱 + Hooks + PAT — AI 개발 운영체제로 가속

오픈AI가 **Codex ChatGPT 모바일 앱 연결**, **Hooks**, **Programmatic Access Tokens(PAT)** 를 동시에 공개했다.

**모바일 앱 연결:**

| 항목 | 내용 |
|------|------|
| 실행 방식 | 스마트폰에서 승인 → 원격 PC/Mac mini/개발 서버에서 실행 |
| 실시간 전달 | 테스트 결과, 터미널 출력, diff, 스크린샷 |
| 지원 연결 | Remote SSH, 기업용 토큰, Hooks |

**Hooks & PAT:**

| 항목 | 내용 |
|------|------|
| Hooks 기능 | 검증 스크립트, 시크릿 검사, 내부 로깅, 저장소별 규칙 자동 삽입 |
| 활용 예시 | 테스트 통과 전 merge 금지, 특정 디렉토리 규칙 자동 적용, 내부 정책 검사 |
| PAT | 기업 운영 로직을 Codex 루프 안에 통합 |

> 포인트: Codex가 단순 코딩 도구가 아니라 개발 조직 안에서 돌아가는 운영 인프라로 변하고 있다. "AI를 자기 조직 흐름에 얼마나 깊게 연결하느냐"가 경쟁력의 핵심이 되는 단계다.

### Figure 03 휴머노이드 로봇 — 30시간 연속 무인 물류 운영

Figure가 **Helix-02 기반 Figure 03** 로봇 3대로 8시간 라이브 데모를 공개하고, 이어 **30시간 연속 운영**까지 달성했다.

| 항목 | 내용 |
|------|------|
| 처리량 | 3만 8천 개 이상 패키지 처리 |
| 운영 시간 | 8시간 라이브 → 30시간 연속 무인 운영 |
| 속도 | 시간당 성능이 사람 작업 속도에 근접 |
| 업계 의미 | 편집된 데모 논란 탈피 → 실제 운영 안정성으로 경쟁 단계 진입 |

> 포인트: 휴머노이드 경쟁은 이제 "움직일 수 있나?"가 아니라 "몇 시간 동안 사람 개입 없이 돈 되는 작업을 유지할 수 있나?"로 넘어가는 단계다.

Moonshot AI의 Kimi가 **Kimi Web Bridge** 확장 프로그램을 공개했다.

| 항목 | 내용 |

### Claude API Pre-warming — TTFT 최대 52% 절감

앤트로픽이 Claude API 응답 속도를 높이는 **pre-warming** 방식을 공개했다.

| 항목 | 내용 |
|------|------|
| 방식 | 시스템 프롬프트를 먼저 전송해 캐시에 올려두고 사용자 요청 즉시 응답 |
| 효과 | 첫 토큰 생성 속도(TTFT) 최대 **52% 절감** |
| 캐시 유지 | 기본 5분 → 연속적인 작업 흐름에 유리 |
| 다중 턴 | 전체 프롬프트 prefix 캐싱으로 latency + 토큰 비용 동시 절감 |

> 포인트: 개발자라면 시스템 프롬프트 pre-warming을 적용해볼 만하다. 다만 Claude Pro/Max에서 토큰 소모가 빠르다는 커뮤니티 불만이 이어지는 상황이라 "비용과 GPU 부담을 사용자 최적화로 넘기는 것 아니냐"는 반응도 있다.

### Qwen3-VL-8B — 사고형 VLM, RTX 4070 구동 가능

**Qwen/Qwen3-VL-8B-Instruct** — Alibaba Qwen 팀, Thinking 버전 포함

| 항목 | 내용 |
|------|------|
| 모드 | Instruct + **Thinking** (추론 과정 포함) 두 버전 |
| 강점 | 텍스트 이해, 시각 인식·추론, 공간·영상 이해, 에이전트 상호작용 |
| VRAM | ~5-6GB (4-bit 양자화 기준) → RTX 4070(12GB) 여유롭게 구동 |
| 라이선스 | Apache-2.0 |
| HuggingFace | [Qwen/Qwen3-VL-8B-Instruct](https://huggingface.co/Qwen/Qwen3-VL-8B-Instruct) |

**Python 안전 탐지 예시:**

```python
from transformers import Qwen2_5_VLForConditionalGeneration, AutoProcessor
from PIL import Image
import torch

model = Qwen2_5_VLForConditionalGeneration.from_pretrained(
    "Qwen/Qwen3-VL-8B-Instruct",
    device_map="auto",
    torch_dtype=torch.bfloat16,
    load_in_4bit=True  # RTX 4070 최적화
)
processor = AutoProcessor.from_pretrained("Qwen/Qwen3-VL-8B-Instruct")

messages = [{
    "role": "user",
    "content": [
        {"type": "image", "image": "작업현장.jpg"},
        {"type": "text", "text": (
            "아래 항목을 체크해 주세요:\n"
            "1. 화재나 연기가 보이나요?\n"
            "2. 작업자가 안전고리(하네스)를 착용하고 있나요?\n"
            "3. 위험 요소가 있다면 구체적으로 설명해 주세요."
        )}
    ]
}]

text = processor.apply_chat_template(messages, add_generation_prompt=True)
inputs = processor(text=text, images=Image.open("작업현장.jpg"), return_tensors="pt").to("cuda")
output = model.generate(**inputs, max_new_tokens=256)
print(processor.decode(output[0], skip_special_tokens=True))
```
