## 1. 모델 출시 & 업데이트

### Claude Opus 4.7 Fast Mode 추가

Anthropic이 **Claude Opus 4.7**에 Fast Mode를 추가했다. 기존과 동일한 성능을 유지하면서 응답 속도를 최대 **2.5배** 높이는 방식이며, Claude Code와 API에서 연구 프리뷰로 제공된다.

| 항목 | 내용 |
|------|------|
| 응답 속도 | 기존 대비 최대 2.5배 향상 |
| 입력 가격 | $30 / 1M 토큰 (기존 대비 6배) |
| 출력 가격 | $150 / 1M 토큰 (기존 대비 6배) |
| 제공 형태 | 연구 프리뷰 (Claude Code, API) |

> 포인트: 속도는 2.5배 빠르지만 비용은 6배 높다. 대규모 자동화 파이프라인에서 레이턴시가 병목인 경우에 적합하고, 일반 개발 용도엔 기존 모드가 유리하다.

---

### Claude Code `/goal` 기능 추가

Anthropic이 Codex의 `/goal` 개념을 Claude Code에 공식 도입했다. 사용자가 완료 조건을 설정하면 Claude가 조건이 충족될 때까지 자율 반복 실행한다.

| 항목 | 내용 |
|------|------|
| 핵심 기능 | 사용자 정의 완료 조건 설정 |
| 동작 | 조건 충족 전까지 Claude 자율 반복 |
| 예시 | "모든 테스트 통과 + lint clean" |
| 연관 기능 | `/loop`, `/schedule`, stop hook |

> 포인트: Codex가 먼저 선보인 '조건 기반 에이전트 루프'를 Claude Code가 흡수했다. 단순 코드 생성기에서 **지속 실행형 에이전트 운영 환경**으로 진화하는 흐름이다.

### MiniCPM-V 4.6 — 최경량 산업 안전 모니터링 추천

**openbmb/MiniCPM-V-4.6** (2026-05-14 기준 Hugging Face 트렌딩 1위, 16.8k 다운로드)

- 기반: SigLIP2-400M 인코더 + Qwen3.5-0.8B LLM
- 파라미터: 1B (Qwen3.5-2B 수준 시각 이해 성능 달성)
- 비주얼 토큰 압축: 4x/16x 선택 가능, 기존 대비 FLOPs 50% 이상 절감
- 배포: vLLM, SGLang, llama.cpp, Ollama 지원
- 파인튜닝: SWIFT, LLaMA-Factory 지원, 소비자급 GPU에서 가능
- 라이선스: Apache-2.0

**벤치마크:**

| 모델 | AAII 점수 | OpenCompass | RefCOCO | OCRBench |
|------|---------|------------|---------|---------|
| MiniCPM-V 4.6 (1B) | **13** | 우위 | 우위 | 우위 |
| Qwen3.5-0.8B | 10 | — | — | — |
| Qwen3.5-0.8B-Thinking | 11 | — | — | — |
| Ministral 3 3B | 11 | — | — | — |

```python
from transformers import AutoModelForImageTextToText, AutoProcessor

model_id = "openbmb/MiniCPM-V-4.6"
processor = AutoProcessor.from_pretrained(model_id)
model = AutoModelForImageTextToText.from_pretrained(
    model_id, torch_dtype="auto", device_map="auto"
)

# 산업 안전 Vision Q&A 예시
messages = [
    {
        "role": "user",
        "content": [
            {"type": "image", "url": "이미지_경로_또는_URL"},
            {"type": "text", "text": "현재 이미지에 화재나 연기가 보이나요? 작업자가 안전고리를 착용하고 있나요?"},
        ],
    }
]
```

### ZAYA1-VL-8B — 8B급 최신 고성능 VLM (2026-05-08 출시)

**Zyphra/ZAYA1-VL-8B** — Zyphra의 ZAYA1-8B LLM 기반, 2가지 혁신적 아키텍처 도입

| 항목 | 내용 |
|------|------|
| 기반 모델 | ZAYA1-8B LLM + Qwen2.5-VL Vision Encoder (ViT) |
| 혁신 1 | **Vision-specific LoRA** — 비전 토큰에만 활성화되는 전용 파라미터 |
| 혁신 2 | **Bidirectional Attention for image tokens** — 이미지 토큰 처리 시 인과적 순서 제거 |
| 학습 데이터 | 오픈 데이터만 사용 |
| 라이선스 | Apache-2.0 |

**벤치마크 (동급 모델 비교):**

| 벤치마크 | ZAYA1-VL-8B | Qwen3.5-4B | Molmo2-4B |
|---------|------------|-----------|---------|
| AI2D | 87.5 | 83.7 | 85.4 |
| ChartQA | 82.2 | 82.4 | 86.1 |
| DocVQA | 92.5 | — | 87.8 |
| OCRBench | 79.8 | 85.3 | 62.0 |
| MathVista | 64.0 | 82.3 | 56.5 |

> 포인트: DocVQA 92.5는 문서 판독(OCR, 계약서, 안전 매뉴얼 인식)에서 강점이다. RTX 4070(12GB)에서 4-bit 양자화로 구동 가능하다.

### Qwen2.5-VL 생태계 현황

Qwen2.5-VL 기반 파인튜닝 모델들이 지속 업데이트 중이며, 산업 안전 용도에 적합한 변형들이 등장하고 있다.

| SenseNova-SI-1.1-Qwen2.5-VL-7B | 7B | 고정밀 비즈니스 비전 |

### 화재·안전 감지 특화 모델 조합 추천

| 목적 | 추천 모델 | GPU 요구사항 | 이유 |
|------|---------|------------|------|
| RTX 3070 (8GB), 빠른 실시간 처리 | **MiniCPM-V 4.6** (1B) | ~2GB VRAM | 최경량, FLOPs 50% 절감, 모바일도 가능 |
| RTX 3070 (8GB), 균형 | **Qwen2.5-VL-3B** (3B) | ~5GB VRAM | 풍부한 파인튜닝 생태계, 커스터마이징 용이 |
| RTX 4070 (12GB), 고정밀 | **Qwen2.5-VL-7B** (7B) | ~5GB (4-bit) | 정밀 문서 이해, 안전 매뉴얼 체크 |
| RTX 4070 (12GB), 최신 고성능 | **ZAYA1-VL-8B** (8B) | ~5GB (4-bit) | DocVQA 92.5, 계약서·문서 판독 강점 |

> 포인트: 화재·연기 감지 + 안전장비 확인 용도라면 **MiniCPM-V 4.6 (1B)으로 실시간 탐지 → Qwen2.5-VL-7B로 상세 분석** 파이프라인이 RTX 3070/4070 환경에서 가장 현실적인 조합이다.
