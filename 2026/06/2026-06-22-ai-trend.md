### 📸 OpenAI ChatGPT Images 2.0 출시 — 이미지 생성 시장 지각변동

- OpenAI, **ChatGPT Images 2.0** (GPT Image 2 기반) 공식 출시

  - 소문자 텍스트, UI 컴포넌트, 미세 스타일 제약 등 세밀한 지시 이행 성능 대폭 향상

- **비교 대상**: Google Gemini Nano Banana 2 (이미지 생성 중 웹 검색 기능 동일 탑재)
- 무료 사용자: 기본(instant) 모드 이용 가능 / Plus·Pro·Business: 추론(thinking) 모드 추가 제공
- 💡 **주목 포인트**: 이미지 생성 시장이 "품질 경쟁"에서 "추론 기반 생성"으로 패러다임 이동. @choi.openai — "Video와 이미지 시장은 찢겼네요"

## 3. 경량 VLM (Vision Language Model) 업데이트

> 용도: 이미지 분석 (화재/연기 감지, 작업자 안전고리·안전모 착용 여부 확인 등)
> 타겟 GPU: RTX 3070 (8GB VRAM) / RTX 4070 (12GB VRAM)

---

---

### 🔍 Qwen3-VL — 이번 주 주목 경량 VLM

- Alibaba Cloud의 최신 멀티모달 시리즈 (2B / 4B / 8B / 32B / MoE 30B·235B)
- **핵심 능력**:
  - 네이티브 256K 토큰 컨텍스트 (텍스트+이미지 혼합 가능)
  - 강화된 순수 텍스트 이해 + 시각적 공간 추론
  - 안전장비 착용 여부, 연기·화염 감지 등 Q&A 형태 지원
- **RTX 3070(8GB) 운용**: 2B/4B 모델은 FP16으로도 여유 있게 가동 가능
- Ollama, LM Studio, vLLM 모두 지원

---

### 🆕 Granite Vision 4.1 4B (IBM) — 문서 이해 특화

- LLM 3.4B + Vision Encoder 0.6B의 경량 구성
- 표 추출, 차트 이해, 핵심-값(KVP) 추출에서 프론티어급 성능 주장
- **Apache 2.0 라이선스** — 상업 활용 자유
- MLX (Apple Silicon) 및 vLLM 지원
- 산업 현장 문서(점검 보고서, 안전 체크리스트) 분석에 적합

---

### 🏗️ G2VLM-2B (InternRobotics, CVPR 2026) — 3D 공간 추론

- CVPR 2026 선정 논문 기반 모델
- **기하학적 공간 추론** + 3D 재구성을 단일 VLM으로 통합
- 작업자 위치 파악, 고소 작업 감지 등 3D 공간 분석에 활용 가능
- 2B로 초경량 — RTX 3070 8GB에서 완전 여유 운용

---

### ✅ 화재·안전 감시 파이프라인 권장 구성 (2026-06-21 기준)

```
[RTX 3070 (8GB) 권장 파이프라인]
Step 1: YOLOv26-fire-detection (실시간 화재/작업자 객체 탐지, ~1-2GB VRAM)
    ↓ 화재/연기/작업자 ROI 크롭
Step 2: Qwen3-VL-4B 또는 Granite 4.0 3B Vision (~3-4GB VRAM)
    ↓ 자연어 질의 → 응답
Step 3: 알림 로직 (임계값 초과 시 알람)

총 예상 VRAM: ~4-6GB (RTX 3070 8GB 여유 운용)

[RTX 4070 (12GB) 고정확도 파이프라인]
Step 1: YOLOv26-fire-detection (~1-2GB)
    ↓
Step 2: Qwen3-VL-8B (~5-6GB) — 최고 정확도 안전장비·화재 판별
    ↓
Step 3: G2VLM-2B 보조 분석 (~2-3GB) — 3D 공간 추론 추가 (선택)
```

**안전 점검 예시 쿼리**:
```python
queries = [
    "현재 이미지에 화재나 연기가 있나요?",
    "현재 작업자는 안전고리(안전벨트)를 착용하고 있나요?",
    "작업자가 안전모를 착용했나요?",
    "화재 위험 요소나 가연성 물질이 노출되어 있나요?",
    "고소 작업 중인 작업자가 있나요?",
]
```

## 5. 오늘의 핵심 인사이트

1. **이미지 생성 시장 = 추론 경쟁 시대**: ChatGPT Images 2.0이 "생각하고 만드는" 모델로 전환. Google Nano Banana와 정면 대결 구도. 단순 품질 우위가 아닌 추론 능력이 이미지 생성의 핵심 변수가 됐다.

2. **ChatGPT 50% 붕괴 — AI 어시스턴트 다극화 시대 공식 개막**: 단 1년 반 만에 65% → 46%로 하락. Gemini와 Claude가 각각 28%, 10%를 가져가며 독점 구도 해소. 특히 Claude의 유료 전환율 13%는 수익화 모델 관점에서 주목할 만한 지표.

3. **GLM-5.2 MIT 오픈소스 — 중국 AI의 오픈소스 전략 유효**: Fable 5 차단으로 상징되는 서방의 AI 접근 제한에 맞서 중국은 오픈소스로 응수. 1M 컨텍스트 + 코딩 벤치마크 최상위권 + MIT 라이선스 — 엔터프라이즈 도입 시 실질적 대안.

4. **GPT-5.6 이번 주 내 출시 임박**: Polymarket 83% 확률 + OpenAI 내부 "의미 있는 도약" 발언 + Mini/Pro 동시 출시 계획 — 이번 주말 전후 발표 가능성 높음. 코딩 도구 생태계에 즉각적 영향 예상.

5. **Alibaba open-code-review — 기업용 AI 코드 리뷰의 실전 검증**: 2년간 수만 명 개발자가 사용한 내부 도구 오픈소스화. Claude Code 대비 F1 1.7배, 토큰 비용 1/5 — 대규모 코드베이스 리뷰 자동화에 실용적.

6. **Qwen3-VL — 경량 VLM 화재·안전 감시 최적 조합 업데이트**: RTX 3070에서 Qwen3-VL-4B + YOLOv26 조합이 현실적 최선. G2VLM-2B는 3D 공간 추론(고소 작업 위치 파악)에 보조 활용 가능.
